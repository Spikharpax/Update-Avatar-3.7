Ecoute

Les paramètres suivants définissent la configuration d'écoute du client Avatar

Paramètre

Type

Valeur par défaut

Description

listen -> current

Boolean

false

Il est possible de faire en sorte que chaque client vérifie si la pièce courante est celle qui a captée et réponde ou pas au déclenchement de l'écoute.


Cette propriété est faite pour éviter que 2 clients proches l'un de l'autre ne se déclenchent avec une demande faite sur l'autre client.

true: Si le client est la pièce courante définie par capteur de présence alors l'écoute est déclenchée.
false: Pas de contrôle.

Avatar vérifie la pièce courante par la variable Avatar.currentRoom

listen -> threashold

Float

0.6

Pour améliorer le résultat de la phrase prononcée et traitée par l'API vers laquelle l'audio a été envoyée, une recherche de la grammaire la plus proche est effectuée par une distance de levenshtein. 

Bien sûr, il faut connaître la grammaire pour pouvoir réaliser cette recherche, par conséquent, elle est effectuée uniquement pour les grammaires connues, soit:

  1. Les grammaires qui stoppent l'écoute avant l'analyse de la règle vocalisée (les propriétés "tts", voir les chapitres ci-dessous, Les propriétés des messages)
  2. Les grammaires de question/réponse (askme)


Valeurs possibles : de 0 à 1
Plus la valeur est basse, plus un texte impréci pourra être associé à une grammaire voisine.

listen -> loop_mode

Boolean

false

Après avoir passé une règle, vous avez 2 possibilités:

  • false: Le mode écoute est stoppé automatiquement après la première règle et vous devez repasser le mot-clé pour le réactiver
  • true: Boucle pour que l'écoute soit continue. Cette fonctionnalité est très pratique pour passer plusieurs règles à la suite sans prononcer systématiquement le mot-clé d'accroche.

listen -> timeout

Integer

10

Une temporisation exprimée en secondes s'active lorsque le mot-clé a été validé par le client et que l'écoute commence.
Après le laps de temps défini dans cette propriété, si aucune règle n'est passée, l'écoute s'arrête automatiquement.

Dans un mode en boucle (voir ci-dessus), la temporisation se réinitialise à 0 automatiquement après chaque règle.

listen -> timeout_record

Integer

0

Une temporisation exprimée en milli-secondes (1000 = 1 seconde) permet d'effectuer une temporisation entre le moment où Avatar vous signifie que l'écoute commence et le moment où la capture de l'audio commence.

Utile si le buffer de capture de l'audio "attrape" une partie de la phrase d'Avatar et la prend pour une règle. Ceci peut arriver si le PC utilisé est lent et peu performant. Ajoutez alors une temporisation (en ms) pour qu'Avatar finisse sa phrase avant que la capture de l'audio commence.

listen -> max_restart

Integer

2

Nombre maximum de redémarrage de l'enregistrement après une incompréhension d'Avatar (défini par un "Recommence..."). 
Après la valeur définie ici, l'écoute est annulée automatiquement et Avatar retourne en mode "Attente".

Utile lorsque le client se déclenche par des faux positifs à répétition par exemple si il y a beaucoup de bruit dans la pièce. Il s'arretera alors tout seul.

listen -> force_mute

Boolean

false

Défini si la propriété listen->max_mute (juste ci-dessous) est active ou non.

false: listen->max_mute n'est pas actif
true: listen->max_mute est actif

listen -> max_mute

Integer

2

Nombre de listen->max_restart possibles enchainés (qui ont arrétés l'écoute).

Après la valeur définie, l'écoute est annulée et le client Avatar se met en mode silence et le client ne répond plus aux commandes vocales passées.

Important: Pour sortir de ce mode silence, il est impératif de redémarrer  le client ou mieux d'avoir un client virtuel (une montre, un téléphone, une interface Web, un bouton poussoir connecté etc... pouvant envoyer une requète HTTP pour le remettre en mode écoute)
