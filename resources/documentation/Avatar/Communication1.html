Communication


Les paramètres suivants définissent la configuration réseau pour la communication avec les clients Avatar Windows


Paramètre

Type

Valeur par défaut

Description

http -> ip

String

"localhost"

L'adresse IP du serveur Avatar.

http -> port

Integer

3000

Le port de communication HTTP avec les clients Avatar.


Normalement, identique au port défini dans le fichier de propriété des clients.

default -> client

String

"Salon"

Le client par défaut si celui-ci est homi dans les actions ou si un plugin a besoin d'un client par défaut.

Récupérable par la variable globale Avatar.currentRoom par développement.

Important: Doit être identique à un nom d'un Avatar client existant sur le réseau.

udp -> port

Integer

2222

Port de communication UDP avec les clients Avatar.


Normalement, identique au port défini dans le fichier de propriété des clients.

default -> mapping

Tableau de String

Aucune

Cette propriété permet de "mapper" les retours vocaux d'Avatar sur un autre client que celui normalement désigné comme pièce courante.

Format:
"mapping" : ["pièce mappée 1,pièce réelle 1","pièce mappée 2,pièce réelle 2"],

Par exemple, supposons ce cas de figure:

  • Un seul Avatar Client disposé entre le "Cuisine" et un "Vestibule" qui permet un dialogue avec Avatar suffisament correct dans ces 2 pièces contiguës.
  • La lumière est gérée automatiquement dans ces 2 pièces par capteurs de présence qui définissent la pièce courante (ainsi, il n'est pas nécessaire de dire systématiquement la pièce si on veut allumer/éteindre la lumière)


En définissant ce paramètre ainsi:

  • "mapping" : ["Vestibule,Cuisine"],

L'action sera exécutée dans la pièce "Vestibule" et les retours vocaux d'Avatar seront redirigés automatiquement vers la "Cuisine" alors même que la pièce courante est le "Vestibule".

Autre exemple avec plusieurs "mapping":

  • "mapping" : ["Escalier,Cuisine","Patio,Salon","Couloir,Chambre"],

default -> reboot_time

String

Aucune

Cette propriété prévient le serveur de l'heure à laquelle les clients Avatar sont redémarrés automatiquement avec, par exemple, une tâche Windows. 
Bien sûr ce n'est pas obligatoire, bien qu'il soit bon de redémarrer les PCs quotidiennement si ils fonctionnent H24.


Comme le serveur peut vous envoyer une notification si un client se déconnecte (voir le chapitre sur les notifications), si un reboot automatique est déclenché, aucune notification ne sera envoyée dans un laps de temps de la propriété notification -> reconnectIn autour de l'heure définie ici.


Format:
"reboot_time" : "HH:MM"

Exemple pour un reboot à 3:30 du matin:
"reboot_time" : "03:30"
